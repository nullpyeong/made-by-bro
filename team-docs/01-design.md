# 01. Design (UX/UI)

> 화면·플로우·브랜드를 일관되게. 신뢰 비즈니스라 "싸 보이지 않는" 게 핵심.

| | |
|---|---|
| 담당 | 김지평 |
| 상태 | 운영 중 |

## 책임 범위
- 디자인 토큰·컴포넌트 일관성 (`docs/design/tokens.css`)
- 화면 플로우 / 정보 구조
- 브랜드 톤 (로고, 색, 타이포)
- 접근성·반응형

## 디자인 원칙
- **깔끔한 단일 포맷 레이아웃.** 과한 장식·이모지 남발 금지 — 신뢰감이 곧 전환율.
- 결제·완청 같은 핵심 플로우는 **클릭 수 최소화**, 막힘 없는 경로 우선.
- 다크모드 지원 (이미 토글 구현됨).

## 지금 결정된 것 (Decided)

- 랜딩 섹션 순서: 히어로 → 신뢰띠 → 원장소개 → 기능 → 통계 → 작동방식 → 인기강의 → 약속 → 요금제 → CTA → 풋터
