# TEAMDOC — made-by-bro 운영 문서

> **여기는 "무엇을·왜 그렇게 정했는가"를 남기는 곳이다.**
> 결정은 여기서 내리고, 기록하고, 그다음부터는 기록을 따른다.
> 코드는 *어떻게* 만드는지를 담고, TEAMDOC은 *무엇을 왜* 만드는지를 담는다.

---

## 왜 이게 필요한가

지금은 사실상 1인(+원장/누나) 체제다. 그래서 8개 파트는 **8명의 사람**이 아니라
**한 사람이 번갈아 끼는 8개의 판단 렌즈**다.

- "지금 이 결정은 어느 렌즈로 보는 건가?" 를 명확히 하면, 한 사람이 내려도 **편향이 줄고**
  나중에 "왜 그때 그렇게 했지?"를 자기 자신에게 설명할 수 있다.
- 사람이 늘어나면, 각 파트 문서가 그대로 **그 역할의 인수인계서**가 된다.

규칙은 단순하다: **말로 정하지 말고 문서로 정한다. 정한 건 따른다. 바꾸려면 새 결정으로 덮는다.**

## 의사결정은 이렇게 남긴다 (ADR)

중요한 결정은 [`decisions/`](decisions/) 폴더에 **번호가 매겨진 한 장짜리 문서**로 남긴다.
ADR = Architecture Decision Record(여기선 기술뿐 아니라 마케팅·운영 결정도 포함).

### 언제 ADR을 쓰나
- 되돌리기 어렵거나 비용이 큰 선택 (기술 스택, 결제사, 가격 정책)
- 나중에 "왜 이렇게 했지?"를 반드시 묻게 될 선택
- 두 파트(렌즈)의 의견이 충돌해서 한쪽을 골라야 할 때

작은 결정(버튼 색, 카피 한 줄)은 ADR 없이 해당 파트 문서의 "결정된 것"에 한 줄로 적으면 된다.

### 쓰는 법
1. [`decisions/_TEMPLATE.md`](decisions/_TEMPLATE.md)를 복사
2. 다음 번호로 파일 생성: `decisions/NNNN-제목.md` (예: `0005-faq-섹션-추가.md`)
3. [`decisions/README.md`](decisions/README.md) 인덱스 표에 한 줄 추가
4. 같은 브랜치/PR에 코드 변경과 함께 커밋

### 상태값
- `채택` — 지금 따르는 결정
- `검토중` — 아직 안 정해짐, 토론 중
- `폐기` — 안 하기로 함
- `대체됨 → NNNN` — 더 새 결정이 덮음 (옛 문서는 지우지 말고 상태만 바꾼다)

> **결정을 바꿀 땐 옛 ADR을 수정하지 말고, 새 ADR을 만들어 옛 것을 `대체됨`으로 표시한다.**
> 그래야 "왜 바뀌었는지"의 역사가 남는다.

## 흐름 요약

```
아이디어/문제
   │
   ├─ 작은 결정 → 해당 파트 문서의 "결정된 것"에 한 줄
   │
   └─ 큰 결정 → decisions/NNNN.md 작성 → 인덱스 등록 → 코드와 함께 커밋
                                                   │
                                          그다음부터는 이 문서를 따른다
```
