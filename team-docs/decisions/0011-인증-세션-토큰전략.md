## 맥락
그동안 API 엔드포인트는 인증이 없어 `userId`를 **바디/파라미터로 직접 받았다**(referrals·rewards·events). 이건 누구나 남의 `userId`로 호출 가능 = IDOR 취약점이고, 0006의 "활성화(첫 강의 수강) → 추천인 보상" / "유료 전환 → CAC"를 **실제 본인 행동에 자동 연결**하려면 호출자가 누구인지 서버가 알아야 한다.

스키마(`user_sessions`)는 이미 인증 모델을 전제한다 — `refresh_hash`(회전/폐기용) 컬럼이 있고 access 토큰 저장 컬럼은 없다. 즉 **"서버사이드 refresh 세션 + 무상태 access 토큰"** 구조가 스키마 레벨에서 사실상 정해져 있었다. 이번에 그 구조를 구현하며 열린 선택(access 토큰 형식·수명·의존성)을 박는다.

## 결정 (제안 — 확정 대기)
**access = JWT(짧은 수명) + refresh = opaque 회전 토큰(user_sessions에 해시 저장).**

- **access token**: JWT(HS256), 수명 15분, payload `{ sub: userId, role }`. 무상태 — 매 요청 검증만, DB 조회 없음. `Authorization: Bearer <jwt>`.
- **refresh token**: `"<sessionId>.<secret>"` 형식. `secret`만 SHA-256 해시해 `user_sessions.refresh_hash`에 저장(원문 미저장). 수명 30일. **refresh마다 secret 회전**(옛 토큰 즉시 무효 = 재사용 탐지), logout 시 세션 row 삭제로 폐기.
- **전역 가드(secure-by-default)**: `JwtAuthGuard`를 `APP_GUARD`로 등록 → 모든 라우트 기본 보호. 비로그인 접근 라우트만 `@Public()`로 명시 opt-out.
- **세션 주입**: `@CurrentUser()`로 가드가 넣은 주체를 꺼내, 기존 `userId` 바디/파라미터를 대체.
- **의존성 추가**: `@nestjs/jwt`. 비밀번호 해시는 기존 `bcryptjs` 재사용.

## 근거
- **스키마 정합** — `user_sessions.refresh_hash`가 이미 refresh-회전 모델을 가리킴. access용 컬럼이 없으니 access는 자연히 무상태(JWT)가 맞다.
- **무상태 access의 성능** — 매 요청 DB 세션조회 없이 서명 검증만. 짧은 수명(15분)으로 탈취 위험 제한, 폐기는 refresh 회전/삭제로.
- **회전 = 재사용 탐지** — 옛 refresh가 다시 오면 401. 토큰 탈취 시 피해 창을 좁힌다.
- **secure-by-default** — 새 엔드포인트가 실수로 열리는 걸 막음. 공개는 의도적으로만(`@Public`).
- **버린 대안**:
  - *opaque access(매 요청 DB 세션 조회)* — `user_sessions`에 access 저장 컬럼이 없고, 무상태 대비 매 요청 조회 비용. 기각.
  - *세션 쿠키(서버 세션 스토어)* — SPA + 정적 랜딩 + 추후 모바일 혼재 구조엔 Bearer 토큰이 단순. 기각.
  - *Passport.js* — 현재 미니멀 스타일에 과함. 얇은 가드로 충분. 기각.

## 결과 / 영향
- **신규 엔드포인트**: `POST /auth/login` · `POST /auth/refresh` · `POST /auth/logout` · `GET /auth/me`. `POST /auth/signup`은 가입 직후 토큰까지 발급(자동 로그인).
- **세션 주입으로 전환된 엔드포인트**(바디 `userId` 제거):
  - `POST /referrals/code`, `POST /referrals/redeem`, `GET /referrals/me`
  - `POST /rewards/activation`, `GET /rewards/me`
- **공개(@Public)**: `health`, `GET /courses`, `GET /offers`, `POST /events`(익명 랜딩 퍼널), `auth/signup·login·refresh·logout`.
- **환경변수**: `JWT_SECRET`(필수, 미설정 시 dev 임시값+경고), `JWT_ACCESS_TTL`(기본 15m).
- **검증**: temp(OneDrive 밖) 빌드 + 실서버 e2e **38/38 PASS** — 가드 401 차단, 세션주입 보상 트리거, refresh 회전·재사용 차단, logout 무효화, 로그인 오류 케이스.

### 남은 후속 (이번 범위 밖, 별도 인증 필요)
- **서버/관리자 엔드포인트는 아직 임시 `@Public`** + 코드에 `TODO(ADR 0011)` 표시: `rewards/conversion`(결제 웹훅), `rewards/grant`, `rewards/:id/redeem`, `events/funnel`, `cohorts/*`. → **결제 웹훅 서명 검증 + 관리자 `role` 가드**가 후속.
- **카카오 OAuth**(스키마 `users.kakao_id`)는 미구현 — 후속 ADR/구현.
- **활성 세션 수 제한**(스키마 주석의 "2기기")은 앱 레벨 미구현.
- `rewards/activation`을 **플레이어 첫 재생에 자동 호출**(본인 토큰)로 연결 — 프론트 후속.
