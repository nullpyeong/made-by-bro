# 0002. 영상 보호 수준 3 (Signed URL + 동적 워터마크)

| | |
|---|---|
| 상태 | ✅ 채택 |
| 날짜 | 2026-06-21 |
| 파트 | Engineering |
| 결정자 | 김지평 |

## 맥락
유료 강의 영상이 유출·재배포되면 매출 직결 손해. 보호 수준을 정해야 한다.
(출처: `design-docs/설계문서.md` — "영상 보호: 수준 3")

## 결정
**Cloudflare Stream + Signed URL + 사용자별 동적 워터마크**로 간다.

## 근거
- Cloudflare Stream이 트랜스코딩 + HLS + Signed URL을 내장, 최저 단가.
- Signed URL로 무단 링크 공유 차단, 동적 워터마크(사용자 식별 정보)로 화면녹화 유출 시 추적 억제.
- 더 높은 수준(DRM)은 100명 규모에 과투자.

## 결과 / 영향
- 영상 재생 시 사용자별 워터마크 오버레이 구현 필요.
- Signed URL 만료·갱신 로직 필요 (Redis로 관리 가능).
