# 0015. 런칭 우선순위 — 실결제 보류, 콘텐츠 먼저

| | |
|---|---|
| 상태 | ✅ 채택 |
| 날짜 | 2026-06-21 |
| 파트 | Product |
| 결정자 | 김지평 |

## 맥락
지금까지 백엔드·프론트를 빠르게 쌓아오면서 결제 토대(ADR 0003 토스, 0008 커머스 모델, 0013 웹훅 서명)까지 일부 구현·설계됐다. 다음 자연 수순으로 **Cart/Checkout 실결제(토스) 시작 플로우**가 후보로 올라왔다.

그러나 이 프로젝트의 본질은 **콘텐츠(강의)가 곧 제품**이고(첫 런칭의 적은 잠식이 아니라 무명 — 0006), 전환을 만드는 건 결제 UX가 아니라 **강의·신뢰 콘텐츠**다. 실결제는 토스 가맹 심사·실사양 대조(0013 유보)·약관/정산(Ops/Legal)까지 물려 있어 **무겁고, 콘텐츠가 비면 결제창이 있어도 팔 게 없다.**

## 결정
**실결제(주문 생성→토스 결제창→정산/환불) 구현을 보류하고, 당분간 콘텐츠 축에 집중한다.**

- 콘텐츠 축 = 강의 커리큘럼/플레이어 실연동, 수강·진도, 원장 신뢰 콘텐츠(0007), 강의 목록·상세의 실데이터화.
- 결제는 **"보류"이지 "폐기"가 아니다** — 토대(0003/0008/0013)는 그대로 두고, 콘텐츠가 채워진 뒤 0006 M3(외부 유료 전환) 시점에 다시 연다.

## 근거
- **콘텐츠가 ROI 1순위** — 후기 0개 런칭에서 전환 제약은 가격·결제가 아니라 신뢰·강의(0004·0005·0006에서 반복 확인). 결제창은 팔 콘텐츠가 있어야 의미.
- **결제는 외부 의존이 큼** — 토스 가맹 심사, 0013의 실서명 사양 대조, 약관·정산·환불 정책(Sales-CS/Legal). 콘텐츠보다 리드타임이 길고 외부에 묶임.
- **0006 타임라인과 정합** — M1 활성화·M2 후기는 결제 없이 진행된다. 실결제가 필요한 건 M3(외부 유료)다. 지금 결제에 시간 쓰는 건 순서가 이르다.
- **이미 안전판 존재** — checkout/cart는 현재 localStorage 데모로 화면은 있고, 웹훅은 fail-closed(시크릿 없으면 전부 거부)라 **보류 상태가 사고로 이어지지 않는다.**

## 결과 / 영향
- **하지 않는 것 (보류)**: 주문 생성 API, 토스 결제창 연동, 결제 성공/실패 콜백 바인딩, 정산·환불 웹훅, 0013 실서명 사양 확정.
- **집중하는 것**: 강의 콘텐츠·플레이어·수강/진도 실연동, 원장 신뢰 콘텐츠(0007), 멤버십/구독 *구조*는 두되 *결제*는 안 붙임.
- **관련 ADR 상태 변화 없음** — 0003·0008·0013은 그대로 유효(구현만 뒤로 미룸). 0013은 계속 🟡(실결제 착수 시 확정).
- **리스크**: 콘텐츠만 쌓고 결제를 너무 늦추면 M3에서 결제 리드타임(가맹 심사 등)이 병목이 될 수 있음 → **M2 후반에 토스 가맹 신청만 미리 시작**하는 것을 후속 체크포인트로 남긴다(구현이 아니라 행정 선행).
