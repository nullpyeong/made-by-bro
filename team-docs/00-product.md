# 00. Product (PM/PO)

> 뭘 만들지 정하고 우선순위를 잡는다. "안 만들 것"을 정하는 게 더 중요하다.

| | |
|---|---|
| 담당 | 김지평 (대표 겸) |
| 상태 | 운영 중 |

## 책임 범위
- 제품 비전 / 타겟 사용자 정의
- 로드맵·우선순위 (무엇을 먼저, 무엇을 안 함)
- 기능 스펙 합의 (Design·Engineering과 조율)
- 출시 여부 판단

## 타겟 & 핵심 가치
- **타겟**: 초·중등 영어 인강 수강생과 학부모
- **핵심 가치**: "끝까지 완주(완청)시킨다" — 완청 인증 + 구간 퀴즈 + 1:1 Q&A
- **차별점**: 같은 원장의 대면 강의를 1/10 가격에, 무약정으로

## 지금 결정된 것 (Decided)
- 제품 본체는 **강의 콘텐츠**, 웹은 그릇. → 초기 리소스는 콘텐츠·마케팅·CS에 우선 배분 ([TEAMDOC README])
- 결제 모델: **단건 구매 + 기간제 수강권(30/90/무제한)** ([설계문서](../design-docs/설계문서.md))

## 열린 질문 (Open)
