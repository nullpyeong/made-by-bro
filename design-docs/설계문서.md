# 인강 사이트 시스템 설계 문서

| 항목 | 내용 |
|---|---|
| 대상 규모 | 동시 100명 / 누적 50~100명 |
| 개발 체제 | 1인 풀스택 |
| 결제 모델 | 단건 구매 + 기간제 수강권(30/90/무제한) |
| 영상 보호 | 수준 3 — Signed URL + 사용자별 동적 워터마크 |
| 물리서버 | 없음 (전부 매니지드/서버리스) |

---

## 1. 기술 스택

| 영역 | 선택 | 근거 |
|---|---|---|
| Frontend | **React + Vite + TypeScript + Tailwind** (`apps/web`) | SPA, 빠른 빌드, 프론트 책임 분리 |
| Backend | **NestJS + TypeScript** (`apps/api`) | 모듈/DI 구조로 결제·수강권·영상보호 도메인 확장에 강함 |
| 공유 | **`packages/shared`** | 프론트·백 공유 DTO/타입 |
| DB | **PostgreSQL** (Supabase / Neon 매니지드) | 관계형 정합성, 단일 인스턴스로 100동접 충분 |
| Cache/Queue | **Redis** (Upstash) | JWT refresh 관리·rate limit·만료배치. 초기엔 최소 사용 |
| Storage | **AWS S3** | PDF 자료·썸네일. 다운로드는 Signed URL |
| Video | **Cloudflare Stream** | 트랜스코딩+HLS+Signed URL 내장, 최저단가 |
| Auth | **JWT** (Access+Refresh) + 카카오 OAuth | RBAC 기반 |
| Admin | **AdminJS** (또는 React-Admin) | 자체구축 대비 개발기간 단축 |
| 결제 | **토스페이먼츠** (포트원 대안) | 국내 PG, 단건/기간제 대응 |
| 배포 | 프론트 **Vercel** / 백엔드 **Railway·Render** 등 | SPA는 정적 배포, API는 서버 인스턴스 |

> 결정 근거는 [ADR 0001](../team-docs/decisions/0001-백엔드-프레임워크.md) 참고. 초기엔 단일 Postgres·최소 Redis로 시작하고, Stream→CloudFront 자체 파이프라인은 확장 시점(수천 명↑)에 점진 이전 가능.

## 2. 시스템 아키텍처

```
[브라우저/모바일웹]
      │ HTTPS
      ▼
[Vercel · React SPA]   (apps/web)
      │ REST API
      ▼
[Railway 등 · NestJS API]   (apps/api)
        │
        ├──▶ PostgreSQL (Supabase)   : 유저/코스/결제/진도
        ├──▶ Redis (Upstash)         : refresh token, rate limit
        ├──▶ S3                       : PDF/썸네일 (Signed URL)
        ├──▶ Cloudflare Stream        : 영상 (Signed URL + 워터마크)
        ├──▶ 토스페이먼츠 API          : 결제 승인/환불
        └──▶ 카카오 OAuth             : 소셜 로그인
```

비기능 요구는 이 구성으로 충족된다. 영상 시작 < 3초는 Stream 글로벌 CDN HLS가, API < 500ms는 Postgres 인덱스 + Redis 캐시가, 100 동접은 NestJS API 인스턴스(필요 시 수평 확장)가 담당한다.

## 4. 핵심 흐름

### 4-1. 영상 접근 검증 (보호수준 3)
```
1. 클라이언트가 강의 재생 요청 (lecture_id)
2. 서버: JWT 검증 → enrollment 확인
     - status='active' AND (expires_at IS NULL OR expires_at > now())
3. 통과 시 Cloudflare Stream Signed URL 발급
     - 만료시간(예: 4시간), 시청자 식별 토큰 포함
4. 플레이어에 사용자 이메일/ID 동적 워터마크 오버레이
5. 미통과 → 403 (is_preview=true 강의는 예외 허용)
```
> 영상 원본 URL은 DB·클라이언트에 절대 노출하지 않는다. `video_uid`만 저장하고 재생 시점에 서명한다.

### 4-2. 결제 → 수강권
```
1. (쿠폰 입력 시) 쿠폰 검증: valid_until, min_amount, max_uses
2. 토스 결제창 호출 → 사용자 결제
3. 승인 콜백: pg_tid 멱등 확인 → payments.status='paid', paid_at
4. enrollments 생성: expires_at = now() + 수강기간 (무제한이면 NULL)
5. 쿠폰 사용 시 coupon_redemptions 기록 + coupons.used_count++
```

### 4-3. 완청 검증 / 수강만료
```
[완청 검증 - 하트비트 기반]
1. 재생 중일 때만 5~10초마다 현재 위치를 progress API로 전송 (seek/일시정지 중 제외)
2. 서버: 전송 구간을 watched_intervals 에 union 누적 → covered_seconds 갱신
3. covered_seconds / duration ≥ completion_threshold/100 → completed=true
4. 부정탐지: 실제 경과시간 < duration / 최대배속(2x) 이면 비정상 플래그
   disable_seek=true 면 max_position 초과 seek 차단
[수강만료 - 일배치]
   expires_at < now() AND status='active' → status='expired'
```

### 4-4. 영상 퀴즈 (게이팅 / 채점)
```
[inline 게이팅]
1. 플레이어가 timeupdate 로 position_sec 도달 감지 → video.pause() + 퀴즈 모달
2. 답 제출 → quiz_attempts 기록. require_correct=true 면 정답 시에만 재생 재개
3. 미정답 cue 앞으로의 seek 차단

[final 채점]
1. 영상 종료 후 final 퀴즈 세트 노출 → 일괄 제출
2. 서버 채점 → quiz_attempts.score 저장, sum(score) 산출
3. 통과 점수(예: 80) 를 수료 조건에 연계 가능
```
> 클라이언트 게이팅은 우회 가능성이 있으나 100명 규모엔 충분하며, 응답·점수는 서버에 기록되어 통계는 신뢰 가능하다.

### 4-5. 수강률 기반 환불 (법정)
```
[학원의 설립·운영 및 과외교습에 관한 법률 시행령 기준]
1. 환불 요청 → 해당 코스 진도율(progress_rate) 계산
2. 구간 판정:
     - 총 강의의 1/3 경과 전  → 결제액의 2/3 환급
     - 1/2 경과 전          → 결제액의 1/2 환급
     - 1/2 경과 후          → 환급 없음
3. refunds 기록(refund_amount) → 승인 → 토스 부분환불 API
   → payments.status = refunded / partially_refunded, enrollments=cancelled
```
> 정확한 환급 비율·기산점은 운영 약관에 명시하고, 위 기본값은 코드 상수로 분리해 정책 변경에 대응한다.

## 5. 폴더 구조 (React + NestJS 모노레포 기준)

```
apps/
├─ web/                    # React + Vite (SPA)
│  └─ src/
│     ├─ pages/            # 코스목록·상세·로그인·수강 플레이어·어드민
│     ├─ components/       # UI (반응형)
│     ├─ lib/api/          # NestJS API 클라이언트
│     └─ stores/           # 상태관리
└─ api/                    # NestJS (REST API)
   └─ src/
      ├─ auth/             # login, signup, kakao, reset (JWT, RBAC 가드)
      ├─ courses/          # list, detail, search
      ├─ lectures/         # stream-url, progress, bookmark
      ├─ payments/         # checkout, confirm, refund (토스 연동)
      ├─ coupons/          # validate
      ├─ stream/           # Cloudflare Stream signing
      ├─ jobs/             # 만료 배치 (Cron 모듈)
      └─ prisma/           # Prisma/Drizzle 클라이언트
packages/
└─ shared/                 # 프론트·백 공유 DTO/타입
```

## 10. 보안 체크리스트

- 비밀번호 argon2id 해시, 카카오 전용 계정은 password NULL
- JWT Refresh 회전 + Redis 블랙리스트(로그아웃/탈취 대응)
